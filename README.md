# WPF MVVM 项目模板

这是一个基于 .NET (WPF) 和 MVVM 模式的桌面应用程序项目模板。旨在提供一个清晰、模块化且易于扩展的项目结构。

## 🚀 特性

- **目标框架**: .NET 8.0 / .NET 10.0 Windows
- **架构模式**: MVVM (Model-View-ViewModel)
- **核心库**:
  - `CommunityToolkit.Mvvm`: 高效、轻量级的 MVVM 库。
  - `MaterialDesignThemes`: 现代化的 Material Design UI 风格。
  - `Microsoft.Extensions.Configuration`: 灵活的配置管理。
  - `Serilog`: 强大的结构化日志记录。
  - `Microsoft.Toolkit.Uwp.Notifications`: Windows 桌面通知支持。

## 📂 目录结构

项目采用分层结构，职责清晰：

- **📂 Converters**: 存放 XAML 值转换器 (IValueConverter)，用于 View 和 ViewModel 之间的数据转换。
- **📂 Models**: 存放数据模型和实体类，代表业务数据。
- **📂 Services**: 存放业务逻辑服务接口及其实现，处理核心业务功能。
- **📂 Src**: 存放通用的辅助类、扩展方法或其他源代码。
- **📂 ViewModels**: 存放视图模型，负责处理 View 的显示逻辑和命令绑定。
- **📂 Views**: 存放 XAML 视图文件 (Windows, UserControls)。

## 🛠️ 快速开始

1. **创建 ViewModel**: 在 `ViewModels` 目录下创建新的 ViewModel 类，继承自 `ObservableObject`。
2. **创建 View**: 在 `Views` 目录下创建新的 Window 或 UserControl。
3. **依赖注入**: 在 `App.xaml.cs` 中注册新的服务和 ViewModel。
4. **运行项目**: 按 `F5` 启动调试。

## 📝 配置

- **template.json**: 模板配置文件位于 `.template.config` 目录。
- **项目属性**: 可以在 `.csproj` 文件中修改项目元数据和依赖项。

## 📦 发布

项目包含自动打包脚本。使用 `dotnet publish` 发布项目后，会自动在发布目录旁生成包含版本号的 ZIP 压缩包。
